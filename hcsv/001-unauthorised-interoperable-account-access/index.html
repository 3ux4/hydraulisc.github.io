HCSV #001 - Unauthorised Interoperable Account Access

This paper goes over the Hydraulisc-Centric Security Vulnerability no.1 (HCSV #001)

HSCV #001, named "Unauthorised Interoperable Account Access", is a major security vulnerability that affects any and all Hydraulisc-powered platforms, including The Hydraulisc App.

This was a SEVERE Security Vulnerability, to which we at Hydraulisc highly recommended you change your password(s)!

HCSV #001 - An in-depth explanation

With HCSV #001, any and all Accounts across any and all Hydraulisc-powered Apps, Websites, etc. were shared publicly.

Specifically, this Security Vulnerability targeted the Hydraulisc Authorisation Authority, the User Manager which handles Users their related User Objects and any other information regarding the specific User (including email, password, profile image, nickname, last seen status, custom status and trees).

We highly suggest you change your password(s)!


HCSV #001 created a server-side cache which logged new users into new connections under the account of the previously logged in user. This alone raises decent concern, as there's a decent amount of damage that can be done by simply being logged in to another user's account (email spoofing, account deletion, password changes, and in some extreme cases, accounts getting banned for violating a platform's rules). As a result, multiple Users, across multiple connections, on various devices, would all be logged into a single account and have full access as if it was their own.

Additionally, HCSV #001 Bypassed Captcha's, which allowed the new User to bypass most Captcha Challenges thrown at them on that specific platform, and Bypassed Password Confirmation prompts, which automatically forced a valid authorisation to the backend, which allowed destructive actions (password changes, account deletions, Enabling/Disabling 2-Factor Authorisation)

HCSV #001 also forced all write operations to the Hydraulisc Database to be valid, excluding some restricted actions.

HCSV #001 - What's been done

First of all, this is (at the time of writing) no longer a concern. The team at Hydraulisc-Centric Security Vulnerability have agreed this Security Vulnerability is no longer reproducible.
